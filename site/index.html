codecricket

Move freely between Claude Code and OpenAI Codex inside a living project, without ever losing your setup.

Most of my projects are Claude-Code-first: a CLAUDE.md and a curated memory store. Codex reads AGENTS.md. codecricket mirrors one into the other (and back), so the same project works in either tool.

  • Byte-exact round-trip. Your CLAUDE.md + memory map into fenced regions of AGENTS.md and come back identical — webhooks, tokens, exact frontmatter and all.
  • Foreign-block safe. Hand-written content and other tools' managed blocks in AGENTS.md are left untouched.
  • Rename-aware. If a folder was renamed and its memory got stranded under the old name, it tells you — and you can attach it from the dashboard.
  • Keeps secrets out of git. Live tokens render so Codex works, but the file is gitignored; it blocks if AGENTS.md is already tracked.
  • Dashboard or plain CLI. An interactive terminal dashboard, or scriptable init / status / sync.
$ git clone https://github.com/fransjorden/codecricket
$ cd codecricket && npm install
$ npm install -g bun   # the dashboard renders via OpenTUI under Bun
$ npm link             # provides the `cricket` command
$ cricket
A personal tool, built for my own workflow and shared as-is. No telemetry, no warranty. It only ever writes AGENTS.md + memory files, each backed up to .bak first. The dashboard shows every change and asks before writing; a project's first CLI sync is a dry-run by default.
